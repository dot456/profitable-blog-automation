# 개발 블로그 자동화 규칙

## 블로그 개요
- **목적**: 개발 관련 수익화 블로그 운영
- **플랫폼**: Astro 기반 정적 블로그
- **테마**: 깔끔한 흑백 미니멀 디자인

---

## 글 작성 규칙

### 트리거
사용자가 **"새 글 작성"** 이라고 말하면 자동으로 새 블로그 글을 작성한다.

### 글 형식
- **포맷**: 마크다운 (.md)
- **저장 위치**: `blog/src/content/blog/`
- **파일명**: 영문 케밥케이스 (예: `react-hooks-guide.md`)

### 글 구조 (Frontmatter)
```yaml
---
title: "SEO 최적화된 제목"
description: "검색 엔진에 노출될 메타 설명 (150-160자)"
pubDate: "YYYY-MM-DD"
heroImage: "Unsplash 이미지 URL"
---
```

### 콘텐츠 요구사항

1. **분량**: 최소 5000자 이상
2. **말투**: AI 느낌 나지 않는 자연스럽고 친근한 말투
3. **주제**: 매번 새로운 개발 관련 주제 선정
4. **이미지**: 글 중간중간 Unsplash 이미지 삽입
   - 형식: `![이미지 설명](https://images.unsplash.com/...)`

### SEO 최적화 체크리스트

- [ ] 키워드가 포함된 매력적인 제목
- [ ] 메타 설명 150-160자 이내
- [ ] H2, H3 등 헤딩 태그 적절히 활용
- [ ] 내부/외부 링크 포함
- [ ] 이미지에 alt 텍스트 작성
- [ ] 자연스러운 키워드 배치
- [ ] 가독성 좋은 문단 구성 (3-4문장 단위)

### 글 구성 권장 패턴

1. **서론** - 독자의 관심을 끄는 도입부, 글을 읽어야 하는 이유
2. **본론** - 핵심 내용을 H2/H3로 구분하여 상세 설명
3. **예제/코드** - 실용적인 코드 예시 포함
4. **결론** - 핵심 내용 요약 및 다음 단계 제안
5. **마무리** - 댓글 유도 또는 관련 글 추천

## 주제 카테고리 (순환 선택)

- 프론트엔드 (React, Vue, Next.js, TypeScript 등)
- 백엔드 (Node.js, Python, Java, Go 등)
- DevOps (Docker, Kubernetes, CI/CD 등)
- 데이터베이스 (SQL, NoSQL, ORM 등)
- 개발 도구 및 생산성
- 소프트웨어 아키텍처
- 알고리즘 및 자료구조
- 커리어 및 취업 팁
- 최신 기술 트렌드

## Unsplash 이미지 가이드

### 검색 키워드 예시
- `coding`, `programming`, `developer`
- `computer`, `laptop`, `workspace`
- `technology`, `software`, `code`
- `office`, `startup`, `team`

### 이미지 URL 형식
```
https://images.unsplash.com/photo-{ID}?w=1200&q=80
```

---

## 주의사항

1. 저작권 문제없는 Unsplash 이미지만 사용
2. 코드 예시는 실제로 동작하는 코드로 작성
3. 최신 정보 기반으로 작성 (구버전 문법 지양)
4. 과도한 광고성 문구 자제
5. 독자에게 실질적인 가치 제공
